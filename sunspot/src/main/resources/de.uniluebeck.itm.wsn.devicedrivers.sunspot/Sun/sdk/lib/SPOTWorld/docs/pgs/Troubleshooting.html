Troubleshooting

If you're having trouble with Solarium, please give the following a try:

1) Make sure you are using the latest release

If you're using an older version, the [TBD link to newest release of Solarium] may already include a fix for your problem.


2) Make sure the SDK is installed correctly

If you're having trouble interacting with SPOTs, make sure that you can execute all of the basic commands outlined in the SPOT Developer's guide. There are command-line equivalents for most Solarium menu items (type ant help in a terminal window for a list of available commands). Make sure that these ant commands work properly.


3) Check for known issues

Here are some of the known issues in this release of Solarium.

  • The Emulator does not currently work if the file path where the Sun SPOT SDK has been installed includes an embedded space character. Windows users might especially encounter this problem as the standard installation location for Windows is C:\Program Files\. The current workaround is to install the SDK in a different location where none of the folders in the file path has a name with an embedded space character.
  • Only simple applications can be paused/resumed or migrated from one SPOT to another because the current framework for saving and restoring an application's state is incomplete. In particular, applications with active network connections cannot be paused/resumed or migrated properly.

If it still doesn't work

Still have your problem? Please report it on the Sun SPOT forums. It could be a known problem for which someone may have already discovered a workaround or a previously unreported issue requiring a new bug report.
